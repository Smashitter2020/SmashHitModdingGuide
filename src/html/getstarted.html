Get Started with Smash Hit Modding

While modding is not officially supported through an in-game editor or SDK, the original developers at Mediocre AB have expressed that they are comfortable with the modding community, provided that mods do not give away paid content (such as the Premium unlock) for free. This permissive stance has allowed a thriving community to emerge.
Recommended Version: v1.4.3 is considered the optimal version of Smash Hit for modding. This is the last version developed by Mediocre AB themselves. Versions 1.5.0 and later were developed by a different organization and introduced changes that make them less suitable for modding. Always source v1.4.3 for your modding base.

Required Tools

Before you begin modding, you will need to set up a collection of community-built tools. The table below summarizes each tool, its purpose, and key notes.

Tool Purpose Notes
Blender 3D modeling software; hosts the Shatter addon Free, open-source. Required for segment editing. Download from blender.org.
Shatter Blender addon for creating and editing segments Primary level editor by knot126. Supports boxes, obstacles, decals, power-ups, water. Exports XML + LitMesh. Use v1.6.5+ for Blender 4.2+, or v1.5.14 for older Blender. Available on Codeberg and GameBanana.
APK Editor Studio Decompile, edit, repack, sign, and install APKs Essential for injecting modified assets into the game APK. Handles signing and optimization. Available for Windows and Linux.
Notepad++ Advanced text editor For hand-editing XML checkpoint files, Lua room scripts, and other text assets.

The Directory Structure

Below is the overall file structure inside the Smash Hit APK's assets/ directory:

assets/
├── levels/              ← Checkpoint XML files
│   ├── intro.xml.mp3
│   ├── night.xml.mp3
│   └── ...
├── rooms/               ← Room Lua scripts
│   ├── intro/
│   │   ├── part1.lua.mp3
│   │   └── part2.lua.mp3
│   └── night/
│       ├── part1.lua.mp3
│       └── ...
├── segments/            ← Segment XML + mesh files
│   ├── basic.xml.mp3
│   ├── basic.mesh.mp3
│   └── ...
├── obstacles/           ← Obstacle Lua scripts
│   ├── scoretop.lua.mp3
│   └── ...
├── game.xml.mp3         ← Levels in order
└── templates.xml.mp3    ← Template definitions
Why .mp3 extensions? Smash Hit uses the .mp3 extension on non-audio files (e.g., .xml.mp3, .mesh.mp3) as a convention. Android's asset compression system leaves .mp3 files uncompressed, which allows the game to memory-map them efficiently. This is not actual MP3 audio data — it is simply an extension trick.
